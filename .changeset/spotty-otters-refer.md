feat: artifact references — `Artifact.add_reference(uri, name=None, checksum=True, max_objects=None)` records external data by URI in the artifact manifest *without* staging any blob, so large or already-durable datasets can be versioned, deduplicated, alias-promoted, and lineage-tracked without copying the bytes. Built-in handlers cover `file://` (stdlib, sha256), `http(s)://` (bundled `httpx`, server `ETag`), and `hf://` (core `huggingface_hub`; LFS sha256 or git blob id), and expand a `file://` directory or `hf://` repo path into one entry per object (capped by `max_objects`). Any other scheme (`s3://`, `gs://`, Azure blob URLs, ...) is supported by registering a custom `ReferenceHandler` via the new `trackio.register_reference_handler` — the artifacts documentation includes complete, copy-pasteable handlers for S3, GCS, and Azure — and raises with an actionable message otherwise. `download()` fetches referenced objects (raising if unreachable, and warning — not failing — when a source's re-probed checksum no longer matches the recorded one); `references` / `get_entry_uri` expose the URIs without downloading. A URI whose query string looks like a signed URL (presigned S3/GCS, Azure SAS) triggers a warning before it is persisted, since manifests may sync to a shared HF Dataset. Logging a reference against a Space/server that predates reference support now fails with a clear upgrade message — both when the old server rejects the entry and when it would silently store a reference as a plain file entry — instead of a bare digest-validation error. Each reference's checksum is stored in the single `digest` manifest field — a sha256 for local/LFS references, the provider ETag for HTTP or object-store references, or the URI itself when no checksum can be obtained (with a warning). Existing file-only manifests remain byte-identical, so their `manifest_digest` values are unchanged.
